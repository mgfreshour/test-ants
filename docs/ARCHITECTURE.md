## 7. Performance Considerations

### 7.1 Target: 10,000 ants at 60 FPS

| Concern | Strategy |
|---|---|
| Per-ant AI | Amortize: only update 1/4 of ants per frame (round-robin). Each ant updates at 15 Hz effective rate. |
| Pheromone grid | Fixed-size 2D array, avoid dynamic allocation. Diffuse/evaporate in batch with SIMD-friendly loops. |
| Spatial queries | Spatial hash grid rebuilt each frame. O(1) neighbor lookups. |
| Rendering | Use Bevy's sprite batching. Ants share atlas texture. Instance rendering for identical sprites. |
| Memory layout | Components stored in archetypal tables by Bevy ECS вҖ” cache-friendly iteration. |
| Pathfinding | No pathfinding! Ants use local pheromone gradient + random walk. Emergent paths, not computed paths. |

### 7.2 LOD for Distant Ants

When zoomed out, distant ants can be:
1. Rendered as single-pixel dots instead of full sprites
2. Updated at reduced frequency (every 4th frame instead of every frame)
3. Grouped into "swarm" entities that represent clusters

### 7.3 Bevy System Parallelism

Bevy automatically parallelizes systems that don't conflict on resource/component access:

```
Parallel group 1:  pheromone_sense | enemy_detection | food_detection
Parallel group 2:  ant_movement | predator_ai
Sequential:        combat_resolution (needs mutable Health access)
Parallel group 3:  pheromone_evaporate | food_respawn | colony_stats
```
